Net Reaper Browser Extension — Privacy Policy

Last updated: March 26, 2026

The Net Reaper browser extension ("the extension") intercepts browser downloads and routes them to the Net Reaper desktop application for managed downloading. It communicates exclusively with a locally installed native messaging host on your computer — it does not send any data to external servers.

What the extension accesses

  • Download URLs — when a download starts, the extension reads the URL so it can forward it to the Net Reaper desktop app on your machine.
  • HTTP request headers — the extension captures the Cookie and Referer headers that your browser already sends for the download request. These are forwarded to the desktop app so it can authenticate with the server on your behalf (e.g. to pass Cloudflare verification).
  • Cookies — used only as a fallback if request headers are unavailable. Cookies are read for the specific download URL only and forwarded to the local desktop app. They are never stored, logged, or transmitted externally.

How data flows

All data captured by the extension is sent exclusively to the Net Reaper desktop application running on your local machine, via Chrome's Native Messaging API. The communication path is:

  1. Extension → Native Messaging Host (local Node.js process)
  2. Native Messaging Host → Named Pipe (local IPC) → Net Reaper desktop app

No data ever leaves your computer. There are no external API calls, servers, or cloud services involved.

What we store

  • Enabled/disabled state — a single boolean stored in chrome.storage.local to remember whether interception is on or off.
  • Debug log — up to 50 recent log entries stored in chrome.storage.local for troubleshooting. These contain timestamps and truncated URLs. The log is visible only in the extension popup and can be cleared at any time. It is never transmitted externally.

What we don't collect

The extension does not collect analytics, telemetry, usage statistics, browsing history, personal information, or any data beyond what is described above. It does not communicate with any external server. It contains no advertising, tracking pixels, or third-party scripts.

Permissions explained

  • downloads — required to detect when a download starts and to cancel the browser's built-in download so Net Reaper can handle it instead.
  • nativeMessaging — required to communicate with the locally installed Net Reaper desktop application via Chrome's Native Messaging API.
  • storage — required to persist the enabled/disabled toggle and debug log entries across browser sessions.
  • cookies — required as a fallback to read authentication cookies for the download URL, so the desktop app can authenticate with the server. Cookies are only read on-demand for the specific download URL and are never stored or sent externally.
  • webRequest — required to capture the HTTP method (GET/POST) and request headers (Cookie, Referer) for download requests, so the desktop app can replay the request with identical authentication.
  • host_permissions (<all_urls>) — required because downloads can originate from any website. The extension only acts when a download event fires — it does not monitor, modify, or intercept regular browsing traffic.

Third-party services

None. The extension does not use any third-party services, SDKs, or libraries.
